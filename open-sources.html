Overview

This page documents open-source software used by the FLAIM Digital Twin and related I2DT deployment environments. It is intended as a license and dependency inventory for the prototype.

Platform Components

Eclipse Ditto

  • Role: Local digital twin middleware for Thing management and event publication
  • License: Eclipse Public License 2.0
  • Link: eclipse.dev/ditto

Eclipse Ditto Python Client

  • Role: MQTT/protocol bindings for Ditto in FLAIM and demo scripts
  • Version: 0.1.0

Docker

  • Role: Container packaging and local execution
  • License: Apache License 2.0

Kubernetes

  • Role: Container orchestration for deployed services
  • License: Apache License 2.0

PostgreSQL

  • Role: Relational storage for platform data
  • License: PostgreSQL License

MongoDB

  • Role: Common backing store option for Eclipse Ditto deployments
  • License: Server Side Public License or vendor-specific terms depending on distribution

FLAIM DT Dependencies

paho-mqtt

  • Role: MQTT client for communication between FLAIM DT and Ditto
  • Version: 2.1.0

requests

  • Role: HTTP client for Ditto REST API calls
  • Version: 2.32.5

python-dotenv

  • Role: Environment configuration for local FLAIM DT deployment
  • Version: 1.2.2

earthdaily-earthone

  • Role: Python client for accessing FLAIM burn and probability products on the EarthOne platform
  • Version: 6.0.1
  • License: Proprietary
  • Link: EarthOne documentation
Note: All dependencies are listed in the flaim-dt/requirements.txt file.

Closed Components

  • FLAIM Digital Twin codebase (this repository) — closed source; authorized access only
  • EarthOne Platform / FLAIM model outputs — proprietary; access via EarthOne account

For EarthOne access: EarthOne documentation.
